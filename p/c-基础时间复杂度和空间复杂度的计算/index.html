时间复杂度和空间复杂度的计算

2024-12-15

算法的时间复杂度和空间复杂度是衡量算法效率的重要指标。

本文将介绍如何计算算法的时间复杂度和空间复杂度。

学习视频链接:https://www.bilibili.com/video/BV1Qs4y1v7AX/

空间复杂度

我们先来介绍一下空间复杂度。要计算空间复杂度，就是看开了多大的数组。一般来说题目会给256MB、128MB、64MB的内存。

以数组这种数据结构为例，我们的int类型是由4个字节组成的，占据了4B的空间，所以大小为1e6的int数组，占据空间为MB（1MB=1024KB，1KB=1024B）。那么64MB的空间就可以开16e6（约莫1e7）个大小的int类型的数组了。

既然如此，我们以int类型为标准，配合大O表示法，就能估算出空间复杂度了。

O(1):常数空间 

1

int sum=0;
for(int i=0;i<100;i++)
{
    sum+=i;
}
cout<<sum<<"\n";

分析：对于这段代码，我们只需要给sum，i分配空间，这种代码所需的内存是极少的，毕竟我们不会手动开一亿多个变量，所以估算需要1*4个字节的空间

O(n):线性空间 

1
2
3
4
5

int arr[n];
for(int i=0;i<n;i++)
{
    arr[i]=i;
}

分析：对于这段代码，我们需要给arr数组分配空间，arr数组的大小为n，在题目中我们把数组开到数据范围的最大值，那么这时候就有n个变量需要分配空间了，总共需要4*n个字节的空间。

O(n^2):二维空间

int arr[n][m];
for(int i=0;i<n;i++)    
{
    for(int j=0;j<m;j++)
    {
        arr[i][j]=i+j;
    }
}

分析：对于这段代码，我们需要给arr数组分配空间，arr数组的大小为nm，在题目中我们把数组开到数据范围的最大值，那么这时候就有nm个变量需要分配空间了，总共需要4*nm个字节的空间。

时间复杂度

一般来说题目会给1s的时间，1s的时间大概能进行5*10^8次运算，但大部分情况只要代码一长就无法做到了。接下来以一些样例来介绍时间复杂度。

嵌套循环类

一般循环，时间复杂度o(n)

{
    doSomething with o(1);
